## 为什么还要自己写博客

社交平台适合快速发布，但不太适合沉淀。  
博客更像一个稳定的个人空间：结构可控、内容可追溯、风格也更接近自己的表达方式。

## 我的目标很简单

- 文章维护尽量轻，只写 `Markdown/MDX`
- 页面足够好看，但不要过度设计
- 部署链路稳定，最好推送代码就能上线
- 项目页可以顺手展示 GitHub 上的作品

## 技术选型

我最后选了 `Next.js + MDX + GitHub Pages`。

```ts
export const stack = {
  framework: "Next.js",
  content: "MDX",
  hosting: "GitHub Pages",
};
```

这套组合最适合“内容优先”的个人博客：静态生成足够快，GitHub Actions 也方便接上部署。

## 页面需要像一本手账

我想让整个站点更轻、更安静一些，所以视觉上会保留大量留白，用低饱和暖色做点缀，尽量让读文章这件事本身成为主角。

### 首页

首页不是单纯的文章列表，而是一个能快速建立个人印象的入口。

### 项目页

项目页要从 GitHub 获取基础数据，但同时允许我自己补中文文案和排序。

## 接下来要继续补的东西

后续我会继续完善：

- 评论系统扩展位
- 更多文章模版
- 更完整的关于页内容
