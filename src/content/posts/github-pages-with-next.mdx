## 先把站点做成全静态

如果目标是 `GitHub Pages`，那就要优先选择静态导出的页面能力。  
这意味着页面数据最好在构建期就准备好，而不是依赖运行时服务。

## Next.js 里要注意什么

最关键的是这几个配置：

```ts
const nextConfig = {
  output: "export",
  trailingSlash: true,
  images: { unoptimized: true },
};
```

这样构建后会直接生成静态文件，便于 Actions 上传到 Pages。

## GitHub Actions 做什么

工作流的职责很清楚：

1. 安装依赖
2. 运行构建
3. 上传 `out` 目录
4. 发布到 `GitHub Pages`

## 为什么我喜欢这条链路

因为它足够直接。  
内容写完、提交代码、推送主分支，然后等待部署完成。

### 成本

免费。

### 维护感受

稳定，且不需要额外维护服务器。

## 一个小提醒

如果未来切换到 `username.github.io/repo-name` 这种子路径部署，就需要额外设置 `basePath`。  
所以我会提前把这层配置留好，避免以后重构。
